# 任务：为独立摄影师构建作品集网站

## 背景

你在为一名独立摄影师开发个人作品集网站。她拍摄两类题材：黑白人像特写，以及高原地区的自然风光与牧场生活记录。她希望网站呈现三个系列：《凝视》（人像）、《无人之境》（风光）、《高原牧歌》（牧场）。

网站需要用 **React + TypeScript + Vite** 搭建。所有真实图片素材和内容数据已经提供在 `mock-data/` 目录下，视觉设计参考、字体文件在 `assets/` 目录下，请直接使用，不要臆造或替换其中的业务数据（照片、标题、文案）。

## 页面结构

1. **首页 `/`**：Hero 区展示摄影师简介，下方展示三个系列的精选预览入口。
2. **作品集页 `/work`**：展示全部照片的网格视图，支持按分类（肖像 / 风光 / 牧野）筛选。
3. **系列详情页 `/work/:seriesId`**：以叙事式长图文排版展示某一系列的完整内容（图文交替、引用摘要），复用与 `/work` 相同的照片数据源。
4. **关于页 `/about`**：摄影师简介、经历时间线。
5. **联系页 `/contact`**：联系表单。

此外，需要一个**全局灯箱（Lightbox）组件**，在任意展示照片的页面中点击照片即可打开，支持左右切换、关闭、显示照片说明文字。灯箱不是一个独立路由，而是可在多个页面复用的共享组件。

视觉设计请参考 `assets/` 目录下的参考图（`reference_*.png`），风格基调：暗色主题、留白充足、衬线标题字体、金色分隔线点缀。参考图用于指导布局结构、间距节奏、组件状态和整体质感，不要求逐像素还原。

## 必须满足的约束（这些约束彼此耦合，请仔细设计数据流和状态管理，不要孤立地实现每一条）

1. **筛选状态需要在导航间保持**：用户在 `/work` 按分类筛选后，点击进入某个系列详情页再返回 `/work`，之前选择的筛选条件应当保持，而不是重置为"全部"。

2. **灯箱的上一张/下一张导航必须限定在当前筛选结果范围内**：如果用户在 `/work` 筛选出"牧野"分类后打开灯箱，灯箱的左右切换只应在"牧野"分类的照片之间循环，而不是遍历全部 14 张照片。这一点在灯箱作为全局共享组件、且筛选状态位于 `/work` 页面内部时容易被忽略，需要显式设计数据传递方式。

3. **图片加载时需要预留正确的宽高比，避免布局抖动（CLS）**：`mock-data/photos.json` 中每张照片都提供了真实的 `width`/`height` 字段，实际像素尺寸差异很大（如某些照片宽高比接近 3:2，另一些接近 2:3）。网格和详情页在图片完全加载前，应根据这些字段预先撑开正确比例的占位空间，而不是等图片加载完成后再让布局跳动。

4. **系列详情页与作品集页共享同一份数据模型，禁止重复硬编码内容**：`/work/:seriesId` 页面展示的照片、标题、说明文字必须来自 `mock-data/photos.json` 中对应 `seriesId` 的照片集合，不能在组件中另外写一份重复的照片列表。

5. **移动端响应式布局需要按参考图切换交互形态**：参考 `assets/reference_work_mobile.png`，`/work` 页面在移动端宽度下，网格需从多列 masonry 布局切换为单列布局，灯箱内的图片说明需要从桌面端的侧边浮层切换为底部信息条。请在浏览器中实际调整视口宽度验证这一切换是否正确触发。

6. **字体必须离线可用，禁止任何对外部字体 CDN 的网络请求**：`assets/fonts/` 目录下已提供本地字体文件（`inter-400-600.woff2`、`playfair-display-600.woff2`、`playfair-display-italic-400.woff2`），项目必须通过本地 `@font-face` 引入这些文件（可参考 `assets/FONTS.md` 中的声明方式），构建后的页面加载过程中不应出现任何指向 `fonts.googleapis.com` 或 `fonts.gstatic.com` 的网络请求。

7. **联系表单需要有完整的校验与反馈状态**：`/contact` 表单在字段为空或格式不合法时需要显示对应的行内错误提示，且在校验未通过时禁用提交按钮；提交成功后（无需真实后端，可用前端模拟）需要显示明确的成功反馈状态，不能停留在原始表单界面无任何变化。

## 验证要求

完成开发后，请在浏览器中实际操作验证以上各项约束是否生效，而不是仅通过阅读代码判断，尤其是约束 2（灯箱在筛选状态下的导航范围）、约束 3（图片加载过程中的布局稳定性）、约束 5（移动端响应式切换）和约束 6（网络请求面板中确认无外部字体请求），这些问题只有在真实交互和真实浏览器环境下才能被观察到。

## 不要做的事

- 不要臆造、替换或删减 `mock-data/photos.json` 中的照片数据、标题或文案。
- 不要将真实照片放入 `assets/` 目录，或将设计参考图放入 `mock-data/` 目录——两者用途不同，请保持现有的目录划分。
- 不要引入 Google Fonts 或其他外部字体 CDN 链接。
